비개발자 × AI 오케스트레이션

여러 AI를 조합해서, 실제 동작하는 서비스를 만듭니다.

개발자가 아닙니다. 하지만 여러 AI 에이전트를 오케스트레이션하면 기획부터 구현, 배포까지 가능합니다. 이 페이지는 그 과정과 성장의 기록입니다.

접근 방식

AI를 도구가 아닌, 팀으로 다룹니다.

여러 LLM을 단순히 나열하는 것이 아닙니다. 각 모델의 강점을 파악하고, 역할을 나누고, 품질과 보안까지 설계합니다.

저는 코드를 쓰는 사람이 아니라, AI에게 올바른 질문을 설계하는 사람입니다. 프롬프트 엔지니어링을 넘어서, 여러 AI 에이전트가 하나의 시스템처럼 동작하도록 기획하고 조율합니다.

단순히 "AI로 만들었다"가 아니라, 왜 이 모델을 선택했는지, 어떻게 품질을 검증했는지, 어디서 실패했고 어떻게 고쳤는지까지 설명할 수 있는 것이 제 역량의 핵심입니다.

⚙

오케스트레이션 설계

Claude의 분석력, ChatGPT의 생성력 등 각 LLM의 특성을 파악하고, 프로젝트 단계별로 최적의 모델을 배치합니다. 하나의 AI가 아닌, AI 팀을 운영합니다.

⊘

품질 검증과 디버깅

AI의 출력을 맹신하지 않습니다. 교차 검증, 엣지케이스 테스트, 출력 일관성 확인까지 — 결과물의 신뢰도를 직접 책임지고 관리합니다.

◈

보안과 데이터 설계

민감 데이터의 흐름을 사전에 설계합니다. API 키 관리, 권한 분리, 데이터 마스킹 등 비개발자도 반드시 알아야 할 보안 원칙을 적용합니다.

↻

반복 가능한 워크플로우

한 번 만들고 끝나는 것이 아니라, 재사용 가능한 프롬프트 체계와 자동화 파이프라인을 구축합니다. 프로세스 자체가 자산이 됩니다.

과정

AI와 함께 만드는 과정.

코드를 직접 쓰지 않습니다. 대신 문제를 정의하고, AI를 조합하고, 결과를 검증하는 과정을 반복합니다.

01

문제 정의.

해결할 문제를 직접 찾고, 어떤 결과물이 필요한지 기획합니다.

02

AI 선택과 조합.

Claude, ChatGPT 등 프로젝트에 맞는 AI를 골라 역할을 나누고 조합합니다.

03

반복과 개선.

결과를 확인하고, 피드백하고, 다시 요청하며 완성도를 높여갑니다.

04

배포와 공유.

실제 동작하는 형태로 배포하고, 누구나 확인할 수 있게 공유합니다.

자주 묻는 질문

자주 묻는 질문.

정말 비개발자인가요?

네, 개발 교육을 받거나 개발 직무를 수행한 적이 없습니다. 여러 AI 에이전트를 활용해 기획부터 구현, 배포까지 진행합니다.

어떤 AI를 사용하나요?

Claude, ChatGPT 등 여러 AI를 프로젝트 특성에 맞게 조합합니다. 하나의 AI에 의존하지 않고, 각 AI의 강점을 활용하는 오케스트레이션 방식으로 작업합니다.

프로젝트를 만드는 과정이 궁금합니다.

문제를 정의하고, 적합한 AI를 선택해 조합하고, 결과를 검증하며 반복 개선한 뒤, 실제 동작하는 형태로 배포합니다. 한 번에 완성하지 않고, 차근차근 발전시킵니다.

대표 프로젝트는 무엇인가요?

실제 업무에서 AI와 자동화로 해결한 사례가 대표 프로젝트입니다. 전자 서명 수집(550명 2주→3일), 경조화환 주문 자동화(휴먼에러 0, 행정 90% 절감), 직무 키워드 분석(EVP 600건+), 그리고 채용공고 자동 생성(ottogi-recruit)입니다.

앞으로의 계획은요?

HTMLvault, mnemo-hook, ottogi-recruit 등 지식 관리와 업무 자동화 도구를 계속 만들어가고 있습니다. AI 오케스트레이션 역량을 넓혀가는 과정입니다.

연락은 어떻게 하나요?

GitHub Issue로 연락할 수 있습니다. 프로젝트 검증, 데모 확인, 협업 제안 모두 환영합니다.
