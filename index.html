작업 방식

1

문제 정리

반복되는 수작업, 필요한 입력, 최종 산출물, 담당자가 확인해야 할 상태를 먼저 정리합니다.

2

흐름 설계

화면, 문서, 저장 위치, 알림, 다운로드까지 실제 업무가 지나가는 순서를 설계합니다.

3

AI 도구 활용 구현

AI 도구를 활용해 웹앱, 자동화 스크립트, 문서 생성 흐름을 빠르게 만들고 직접 다듬습니다.

4

동작 확인

핵심 링크, 파일 생성, 외부 연동, 공개 가능한 정보 범위를 실제 실행 기준으로 확인합니다.

5

배포와 운영

Vercel, GitHub Pages, 공개 저장소처럼 확인 가능한 형태로 배포하고 이후 수정 흐름까지 관리합니다.

직접 만들고 운영하는 것

실제로 동작하는 4가지 사례입니다. 각 사례는 문제, 해법, 내 역할, 상태를 기준으로 정리했습니다.

HR 온보딩 자동화 시스템

신규 입사자 온보딩 서류를 전자서명·PDF·Sheets·이메일 흐름으로 자동화한 HR 운영 시스템입니다.

  • 문제: 신규 입사자 온보딩에서 전자서명, PDF 생성, 진행 상태 기록, HR 담당자 이메일 발송을 수동으로 처리해야 했습니다.
  • 해법: 7종 HR 문서에 전자서명을 받고, PDF를 생성한 뒤, Google Sheets에 상태를 기록하고 이메일 발송까지 이어지는 웹앱 워크플로우를 만들었습니다.
  • 내 역할: 문제 정의, 화면 흐름, 문서 처리 로직, Google 연동, 이메일 발송, 배포까지 전체 흐름을 설계하고 구현했습니다.

  • 해법: Google Sheets 데이터를 기반으로 채용공고 정보를 불러오고, 포스터 화면에서 제목과 로고를 조정한 뒤 PNG로 다운로드할 수 있는 웹앱을 만들었습니다.
  • 내 역할: 채용공고 생성 흐름, Google Sheets 연동, 포스터 화면, PNG 다운로드 흐름을 설계하고 구현했습니다.

AI 도구 운영 대시보드

여러 AI 도구의 작업을 한 화면에서 추적하고, 외부에 공개 가능한 부분만 따로 보여주는 운영 대시보드입니다.

  • 문제: AI 도구를 여러 개 써서 작업할 때, 어떤 도구가 어떤 역할을 맡고 있고 어떤 작업이 진행 중인지 한눈에 보기 어려웠습니다.
  • 해법: GitHub Issues/Projects를 작업 보드로 쓰고, 공개 가능한 정적 화면만 GitHub Pages mirror로 배포하는 Hermes 운영 대시보드를 만들었습니다.
  • 공개 범위: 원본에는 개인 작업 맥락과 운영 메모가 포함될 수 있어 private으로 유지하고, 공개 가능한 정적 화면만 민감 정보 스캔 후 public Pages mirror로 배포합니다.
  • 내 역할: 작업 방식 설계, Codex 구현 지시, 검증 기준 수립, 배포 전 동작 확인, 민감 정보 스캔, 공개 미러 배포와 라이브 검증까지 관리했습니다.
